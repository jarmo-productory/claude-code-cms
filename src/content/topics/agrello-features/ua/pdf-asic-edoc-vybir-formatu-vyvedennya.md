Agrello підтримує три формати виведення електронного підпису: PDF (PAdES), ASIC (XAdES) і EDOC. Кожен формат слугує різним потребам. Вибирайте на основі Ваших вимог підпису, структури файлу та регіональної відповідності.

## Огляд форматів

![Виберіть формат виведення документа](../../../assets/downloaded/documents/choose-whether-your-document-should-be-formatted-as-a-pdf-or.webp)

**PDF (PAdES)** вбудовує підписи безпосередньо у Ваш PDF-файл. Підписаний документ виглядає як звичайний PDF із полями підпису, видимими всередині. Будь-хто з програмою для читання PDF може відкрити його.

**ASIC (XAdES)** створює контейнер — як архів ZIP — який містить Ваші файли та зберігає підписи окремо. **ASIC автоматично примушує до кваліфікованих електронних підписів (QES)** і підтримує декілька типів файлів.

**EDOC** працює як ASIC, але відповідає специфічним регуляторним вимогам Латвії. Використовуйте його для подань латвійському уряду та інституційних контрактів.

## Формат PDF (PAdES)

Формат PDF вбудовує підписи, використовуючи стандарт PAdES (PDF Advanced Electronic Signatures). Підписи з'являються як візуальні поля у Вашому документі.

**Можливості:**

- Один PDF-файл із вбудованими підписами
- Візуальні поля підпису, що показують деталі підписанта
- Підтримує всі рівні підпису: SES, AES, QES
- Заповнювані поля з синтаксисом {{curly_brackets}}
- Текстові поля PDF для збору даних
- Універсальна сумісність — працює будь-яка програма для читання PDF

**Найкраще для:**

- Однофайлових контрактів і угод
- Документів, що потребують видимих полів підпису
- Ситуацій, що вимагають гнучкості підпису (різні підписанти використовують різні методи)
- Одержувачів без спеціалізованого програмного забезпечення

**PDF залишається широко бажаним** для стандартних контрактів завдяки своїй універсальній сумісності на всіх платформах і пристроях.

![Валідація підписів PDF](../../../assets/downloaded/documents/validate-pdf-signatures.webp)

**Обмеження:**

- Підписує лише один PDF-файл на документ
- Не може підписувати інші типи файлів (Word, Excel, зображення)
- Немає автоматичного примушення QES — підписанти вибирають свій метод

## Формат ASIC (XAdES)

ASIC створює файл контейнера (формат ASiC-E), використовуючи XAdES (XML Advanced Electronic Signatures). Файли та підписи залишаються окремими всередині контейнера.

**Можливості:**

- Декілька файлів в одному контейнері (будь-який тип файлу)
- Примушення лише до QES — немає нижчих рівнів безпеки
- Заповнювані поля {{curly_brackets}} у всіх файлах
- Довгострокове архівування зі стандартом XAdES-A
- Відповідність EU eIDAS
- Підписи зберігаються в папці META-INF

**Найкраще для:**

- Контрактів з додатками (основна угода + експонати + додатки)
- Файлів, що не є PDF (архіви ZIP, зображення, таблиці, технічні креслення)
- Обов'язкових вимог QES
- Довгострокового зберігання документів (десятиліття дійсності)
- Запобігання помилкам методу підпису

**Обмеження:**

- Вимагає сумісної програми перегляду ASIC (не стандартних програм для читання PDF)
- Немає візуальних підписів всередині документів
- Лише QES — не можна використовувати прості або розширені підписи
- Немає текстових полів PDF (лише {{curly_brackets}})

![Деталі сертифіката](../../../assets/downloaded/documents/show-certificate.webp)

## Формат EDOC

EDOC ("Electronically Certified Document") — це національний формат електронного підпису Латвії. Побудований на тій же технології ASiC-E + XAdES, що і ASIC.

![Формат EDOC](../../../assets/downloaded/documents/e-signature-edoc-format-signing-digitally.webp)

**Що робить EDOC іншим:**

- Покращена валідація для латвійської регуляторної відповідності
- Обов'язковий для подань латвійському уряду
- Додаткові функції довгострокової валідації
- Кваліфіковані часові позначки та перевірки відкликання сертифікатів

**Коли використовувати:**

- Подання документів латвійським урядовим службам
- Бізнес-контракти, що вимагають латвійської юридичної відповідності
- Офіційні документи для латвійських інституцій

**Технічні можливості:** Такі ж, як ASIC (багато файлів, лише QES, будь-який тип файлу, підписи XAdES).

## Вибір правильного формату

**Виберіть PDF, коли:**

- У Вас є один файл для підпису
- Вам потрібні візуальні поля підпису всередині документа
- Одержувачам потрібна універсальна сумісність
- Ви хочете гнучкість підпису (дозволити різні методи для різних підписантів)
- Ви працюєте зі стандартними контрактами, угодами або формами

**Виберіть ASIC, коли:**

- Ваші контрагенти — естонські компанії або установи публічного сектора
- Вам потрібно підписати декілька файлів разом
- Ви підписуєте типи файлів, що не є PDF
- QES є обов'язковим для всіх підписантів
- Довгострокове архівування важливе
- Відповідність ЄС критична
- Ви хочете запобігти вибору користувачами неправильних методів підпису

**Виберіть EDOC, коли:**

- Ви подаєте латвійському уряду або установам
- Ваш бізнес має латвійські регуляторні зобов'язання
- Вам потрібна покращена валідація для латвійської юридичної системи

## Сумісність методів підпису

**PDF підтримує всі рівні:**

- Простий електронний підпис (SES) — Agrello e-Sign (намальований/набраний)
- Розширений електронний підпис (AES) — eParaksts mobile
- Кваліфікований електронний підпис (QES) — Smart-ID, Mobile-ID, eID картки

**ASIC підтримує лише QES:**

- Smart-ID
- Mobile-ID
- eID картки (естонські, бельгійські, фінські тощо)
- eParaksts (латвійський eID)

**EDOC підтримує лише QES:**

Так само, як ASIC — Smart-ID, Mobile-ID, eID картки, eParaksts.

**Змішані сценарії:** Якщо деяким підписантам потрібен PDF (візуальні підписи), а іншим потрібен ASIC (декілька файлів), створіть окремі документи для кожного формату. Ви не можете змішувати формати в одному виведенні документа.

## Чому вибір формату важливий

**Юридична чинність:** Усі три формати створюють юридично обов'язкові підписи. Вибирайте на основі вимог Вашої галузі та регіональних правил.

**Ефективність робочого процесу:** Правильний формат зменшує помилки та прискорює підписання.

**Довгострокова відповідність:** ASIC і EDOC забезпечують посилені гарантії архівування з валідацією XAdES-A, підтримуючи дійсність підпису десятиліттями навіть після закінчення терміну дії сертифікатів.
