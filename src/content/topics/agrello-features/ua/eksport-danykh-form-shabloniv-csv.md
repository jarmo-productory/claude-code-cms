Збір даних через документи електронного підпису — це половина цінності. Справжня ефективність походить від автоматичного отримання цих даних у Ваші бізнес-системи. Експорт CSV робить це можливим.

## Чому експорт CSV важливий

Традиційні робочі процеси документів втрачають дані. Хтось підписує трудовий договір. Відділ кадрів вручну вводить їхнє ім'я, дату початку, зарплату та відділ у HRIS. **50 нових працівників означає 50 ручних введень** з гарантованими друкарськими помилками.

**Експорт CSV усуває перенабір.** Ваш шаблон збирає дані через заповнювані поля. Ви завантажуєте відповіді як таблицю. Ви імпортуєте цю таблицю у Вашу CRM, HRIS або аналітичну платформу. Нульове ручне введення.

## Як працює експорт CSV

**Створіть шаблони із заповнюваними полями.** Використовуйте синтаксис {{curly_brackets}} у Ваших шаблонах:

- `{{employee_name}}`
- `{{start_date}}`
- `{{department}}`
- `{{salary}}`
- `{{emergency_contact}}`

![Шаблон із змінними](../../../assets/downloaded/templates/template-with-variables.webp)

**Підписанти заповнюють інформацію.** Коли хтось підписує Ваш документ, вони вводять дані в ці поля перед підписанням.

**Система фіксує відповіді.** Кожне значення поля {{field}} зберігається у Вашому робочому просторі Agrello з підписаним документом.

**Експортуйте в CSV.** Перейдіть до Вашого шаблону. Натисніть "Завантажити результати форми як CSV". Отримайте файл таблиці.

**Імпортуйте в бізнес-системи.** Завантажте CSV у Salesforce, HubSpot, BambooHR, Excel або будь-яку систему, що приймає імпорт CSV.

## Експорт даних форми

**Отримайте доступ до Вашого шаблону.** Знайдіть шаблон у Вашому робочому просторі Agrello, який зібрав дані, що Вам потрібні.

**Завантажте CSV.** Натисніть "Завантажити результати форми як CSV" або подібну опцію експорту.

**Отримайте файл.** Ваш браузер завантажує файл .csv з усіма відповідями форми.

**Вимога дозволу:** Лише користувачі з роллю Редактора можуть завантажувати експорти CSV. Це захищає чутливі дані від несанкціонованого доступу.

## Структура файлу CSV

CSV містить один рядок на підписаний документ і один стовпець на заповнюване поле.

![Структура даних CSV](../../../assets/downloaded/templates/csv-with-contcs.webp)

**Заголовки стовпців** відповідають Вашим {{field_names}}:

```
employee_name, start_date, department, salary, emergency_contact, signer_email, completion_date
```

**Рядки даних** показують відповіді:

```
John Smith, 2025-02-01, Engineering, 95000, jane.smith@example.com, john@company.com, 2025-01-17
Sarah Jones, 2025-02-15, Marketing, 85000, mike.jones@example.com, sarah@company.com, 2025-01-18
```

**Автоматичні поля включені:**

- Адреса електронної пошти підписанта
- Дата завершення документа
- Назва документа
- Часова позначка подання

**Експортуються лише підписані документи.** Документи в стані Чернетки або На підписі не з'являються в CSV — лише повністю завершені угоди.

## Повний приклад: інтеграція адаптації відділу кадрів

Цей приклад демонструє повний робочий процес експорту CSV від створення шаблону до інтеграції HRIS.

**Налаштування шаблону**

Створіть шаблон трудового договору з цими заповнюваними полями:

- `{{employee_name}}` - Повне ім'я
- `{{start_date}}` - Дата початку роботи
- `{{position}}` - Посада
- `{{department}}` - Відділ
- `{{salary}}` - Річна компенсація
- `{{manager}}` - Звітує перед
- `{{emergency_contact}}` - Контактна інформація для екстрених випадків

**Ручний робочий процес (старий спосіб):**

1. Надіслати трудовий договір
2. Співробітник підписує
3. Відділ кадрів завантажує підписаний документ
4. Відділ кадрів відкриває систему HRIS
5. Відділ кадрів вручну вводить усі дані співробітника в систему
6. Відділ кадрів створює заявку на обліковий запис електронної пошти
7. Відділ кадрів створює замовлення обладнання

**Автоматизований робочий процес (з експортом CSV):**

1. Співробітник підписує договір і заповнює всі поля шаблону
2. Експортуйте CSV з Agrello (натисніть "Завантажити результати форми як CSV")
3. Імпортуйте CSV у HRIS (BambooHR, Workday, ADP)
4. Зіставте стовпці CSV з полями HRIS під час імпорту
5. HRIS автоматично створює запис співробітника
6. HRIS ініціює створення облікового запису електронної пошти через інтеграцію
7. HRIS ініціює замовлення обладнання через робочий процес

**Зіставлення CSV для імпорту HRIS:**

- `employee_name` → Поле повного імені
- `start_date` → Поле дати початку
- `position` → Поле посади
- `department` → Поле відділу
- `salary` → Поле компенсації
- `manager` → Поле звітування
- `emergency_contact` → Поле контакту для екстрених випадків

**Заощаджений час:** 40 хвилин на співробітника. З 50 новими працівниками це 33 години усуненого ручного введення даних — плюс нуль друкарських помилок.

![Робочий процес Excel і Word](../../../assets/downloaded/getting-started/create-contracts-easily-using-word-templates-and-excel-data.webp)

**Цей же робочий процес застосовується до інших бізнес-систем:** CRM платформи (Salesforce, HubSpot) для захоплення потенційних клієнтів, аналітичні інструменти (Excel, Tableau) для звітності або будь-яка система, що приймає імпорт CSV.

## Розширена інтеграція: API і Zapier

**Інтеграція API:** Розробники можуть використовувати API Agrello для автоматичного експорту даних CSV, коли документи досягають статусу "Підписано". Це забезпечує потік даних у реальному часі в бізнес-системи.

**Робочі процеси Zapier:**

1. Тригер: Новий підписаний документ в Agrello
2. Дія: Експортувати дані форми
3. Дія: Створити/оновити запис CRM
4. Дія: Надіслати повідомлення Slack команді

**Автоматизація без коду** робить інтеграцію доступною для нетехнічних команд.

## Найкращі практики

**Стандартизуйте назви полів.** Використовуйте послідовне іменування {{field}} у всіх шаблонах. `{{company_name}}` скрізь, а не іноді `{{company}}` або `{{company_legal_name}}`.

**Документуйте Вашу схему.** Підтримуйте таблицю, що перелічує всі назви полів і їх значення. Це допомагає товаришам по команді створювати сумісні шаблони.

**Тестуйте імпорти перед запуском.** Створіть тестові документи із зразковими даними. Експортуйте CSV. Імпортуйте у Вашу цільову систему. Перевірте, що всі поля зіставляються правильно.

**Плануйте регулярні експорти.** Не чекайте до кінця року для експорту даних. Щотижневі або щомісячні експорти запобігають накопиченню і забезпечують своєчасну доступність даних.

**Очищуйте Ваші дані.** Переглядайте експорти CSV перед імпортом, щоб виявити друкарські помилки, проблеми форматування або неповні відповіді.

**Валідуйте після імпорту.** Після імпорту в CRM/HRIS вибірково перевіряйте записи, щоб переконатися, що дані передалися правильно.

**Архівуйте експорти CSV.** Зберігайте копії експортованих CSV для аудиторських слідів і резервного копіювання.

## Поширені проблеми

**Відсутні дані в CSV:** Поле було опціональним, і підписант залишив його порожнім. CSV показує порожню комірку. Ваш процес імпорту повинен витончено обробляти порожні значення.

**Невідповідності формату дати:** Підписанти вводять дати в різних форматах (MM/DD/YYYY проти DD/MM/YYYY). Стандартизуйте введення дати з чіткими мітками полів або використовуйте поля вибору дати.

**Спеціальні символи порушують імпорт:** Коми в текстових полях можуть порушити структуру CSV. Переконайтеся, що Ваш експорт використовує правильне цитування CSV. Agrello обробляє це автоматично.

**Дублікатні імпорти:** Імпорт того самого CSV двічі створює дублікатні записи. Використовуйте унікальні ідентифікатори (адреси електронної пошти) для виявлення та злиття дублікатів.

## Коли використовувати експорт CSV

**Високооб'ємні стандартизовані робочі процеси:** Адаптація, захоплення потенційних клієнтів, реєстрація на заходи — будь-який сценарій з повторюваним збором даних.

**Регулярні потреби звітності:** Щотижневі метрики, щомісячне відстеження відповідності, щоквартальний аналіз.

**Міжплатформний потік даних:** Переміщення даних із контрактів у CRM, HRIS, бухгалтерію або аналітичні інструменти.

**Вимоги аудиту:** Ведення записів про те, хто підписав що і коли, що можна експортувати для перевірки відповідності.
